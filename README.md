## 1. 目的
TypeScriptを用いた開発フローの習得と、ユーザーインターフェースにおける「状態（State）」の管理を理解する

## 2. ターゲット
Webブラウザで動作するレスポンシブな電卓アプリ

## 3. 主要機能
### 3.1 基本計算機能
- 四則演算（＋、ー、×、÷）
- 整数および小数点を含む計算
- 「＝」ボタンによる計算結果の表示
- 「AC（オールクリア）」ボタンによる初期化
### 3.2 UI/UX
- 現在の入力値または計算結果を表示するディスプレイ
- ボタン配置：グリッドレイアウト（4列５行程度）
- ホバー時やクリック時のフィードバック（CSSアニメーション）

## 4. 技術スタック
- 言語：TypeScript
- 環境：Vite（高速ビルドツール）
- デプロイ：将来的にクラウド（AWS S3/CloudFront等）へのホスティングを想定

## 5. 【重要】考慮すべきエッジケース（セキュリティ・堅牢性の観点）
フルスタックエンジニアとして、以下の「バグが起きやすいポイント」を仕様に盛り込みます。
- ゼロ除算：10 ÷ 0を行った際に「Error」と表示させるか。
- 桁溢れ：ディスプレイの枠を超える大きな数字の扱い（指数表記にする等）
- 連続演算：1 + 2 + 3 …と入力した際、逐次計算するか最後に計算するか
- Evalの禁止：eval()関数はセキュリティリスク（コードインジェクション）があるため、自作のパースロジックまたはライブラリを使用する。

## 6.　Phase1のゴール
1. TypeScriptで型定義を行い、計算ロジックを分離して実装する
2. GitHubにコードをプッシュし、コード管理の習慣をつける。
